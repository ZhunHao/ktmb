# site/

One-page demo for the [ktmb](../README.md) library, deployed to **Deno
Deploy²** at <https://ktmb-demo.zhunhao.deno.net/>.

## Files

- `index.html` — Apple-inspired one-pager (Figma hand-off design).
- `ktmb-demo.js` — runtime; first-paints from the bundled JSON
  snapshots in `data/`, then polls `/v1/realtime/vehicles` every 6 s
  for live vehicle positions.
- `data/` — generated by [`scripts/build-snapshot.ts`](../scripts/build-snapshot.ts).
  Gitignored. The Deno Deploy build step is the source of truth.

## Deployment

The demo and the live REST API at `/v1/*` ship from the same origin.
[`bin/ktmb-deno.ts`](../bin/ktmb-deno.ts) bootstraps the same Hono app
as `ktmb-api`, then mounts `serveStatic` for `site/` on the same
process. [`deno.json`](../deno.json) declares the install/build/runtime
steps for Deno Deploy. Deno Deploy's GitHub source integration
auto-builds on every push to `main`.

## Local preview

```bash
pnpm install
pnpm snapshot                   # writes data/*.json
deno task deploy:dev            # serves on http://localhost:8000
```

Without a snapshot, the page loads but the snapshot-fed tiles stay
empty because `fetch('./data/*.json')` returns 404. The realtime tile
still works against `/v1/realtime/vehicles` once the server is up.

## Snapshot shape

| File                  | Type                                  | Notes                                           |
| --------------------- | ------------------------------------- | ----------------------------------------------- |
| `meta.json`           | `{ builtAt, calendarWindow, … }`      | Build timestamp, calendar window, vehicle count |
| `stations.json`       | `Array<Station & { lat, lon }>`       | All GTFS stops, augmented with coordinates       |
| `komuter-lines.json`  | `Array<{ id, name, color, stations }>`| Lines with their station sequences              |
| `komuter.json`        | `Record<"line\|station\|date", …>`    | Pre-computed timetables (today only)            |
| `schedules.json`      | `Record<"from\|to\|date", …>`         | ETS/Intercity/Tebrau, top-25 hubs only          |
| `realtime.json`       | `Array<VehiclePosition + x, y, kind>` | First-paint snapshot; replaced by live polling  |

The schedule index is bounded to the busiest 25 stations by trip count
to keep the JSON well under 500 KB.

## Why a snapshot + live polling?

The bundled `realtime.json` snapshot is pre-computed at build time so
the page first-paints instantly from the CDN edge. After load, the
demo polls `/v1/realtime/vehicles` every 6 s and replaces the dots
with actually-live positions. This degrades gracefully on plain static
hosting (where the endpoint 404s and the snapshot stays in place), but
on Deno Deploy the realtime tile is genuinely live.
