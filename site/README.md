# site/

Static GitHub Pages demo for the [ktmb](../README.md) library.

## Files

- `index.html` — Apple-inspired one-pager (Figma hand-off design).
- `ktmb-demo.js` — runtime; fetches the JSON snapshots in `data/`.
- `data/` — generated by [`scripts/build-snapshot.ts`](../scripts/build-snapshot.ts).
  Gitignored. CI is the source of truth.

## Live demo

Deployed at <https://zhunhao.github.io/ktmb/> by
[`.github/workflows/pages.yml`](../.github/workflows/pages.yml). The
workflow runs the snapshot script against live `data.gov.my` GTFS feeds
on every push to `site/**`, on a daily 03:00 MYT cron, and on manual
dispatch.

## Local preview

```bash
pnpm snapshot      # generate data/*.json
npx serve site     # any static server works
```

Without a snapshot, the page loads but the tiles stay empty because
`fetch('./data/*.json')` returns 404.

## Snapshot shape

| File                  | Type                                  | Notes                                           |
| --------------------- | ------------------------------------- | ----------------------------------------------- |
| `meta.json`           | `{ builtAt, calendarWindow, … }`      | Build timestamp, calendar window, vehicle count |
| `stations.json`       | `Array<Station & { lat, lon }>`       | All GTFS stops, augmented with coordinates       |
| `komuter-lines.json`  | `Array<{ id, name, color, stations }>`| Lines with their station sequences              |
| `komuter.json`        | `Record<"line\|station\|date", …>`    | Pre-computed timetables (today only)            |
| `schedules.json`      | `Record<"from\|to\|date", …>`         | ETS/Intercity/Tebrau, top-25 hubs only          |
| `realtime.json`       | `Array<VehiclePosition + x, y, kind>` | One snapshot moment, with 800×480 viewBox coords |

The schedule index is bounded to the busiest 25 stations by trip count
to keep the JSON well under 500 KB.

## Why static, not live?

The library is the API; the demo is a marketing surface. A static site
on Pages is free, has no backend to maintain, and works under any
CDN. The 24-hour staleness is acceptable because GTFS-Static itself
publishes once per day, and GTFS-Realtime vehicle positions are a
stylized "snapshot" rather than a live tracker. If you need
sub-second freshness, run the REST server (`pnpm build && node
dist/bin/ktmb-api.js`) and point a custom client at it.
